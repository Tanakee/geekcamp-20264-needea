# 「NEEDEA」トップ画面：詳細構成・機能仕様書
## 1. グローバルナビゲーション (Global Navigation Bar)
画面最上部に固定（Sticky）される、サービス全体の操作ハブです。
 * **配置 (左端):** ロゴテキスト NEEDEA。
 * **配置 (中央):** 4つのナビゲーションメニュー。
   * Top page / Idea / Builders / Profile
 * **視覚的効果（重要）:**
   * **カレント表示:** 現在開いているページ（例：Top page）の下に太いアンダーラインを引き、ユーザーが現在地を把握できるようにする。
   * **インタラクション:** 選択すると、それぞれの該当ページへスムーズに遷移する。
## 2. ヒーローセクション ＆ 認証 (Hero & Auth Section)
ユーザーを惹きつけ、ワンクリックで参加させるためのエリアです。
 * **メインコピー:** 中央に大きく あなたの不満は、誰かのコードになる を配置。
 * **認証ボタン (Social Auth):**
   * apple でログイン ボタン
   * google でログイン ボタン
   * **配置:** 中央揃えで2つ並べる。
   * **機能:** Google/AppleのOAuth認証を実行。ログイン後はこのボタンが非表示になり、ユーザー専用のコンテンツが表示されるトリガーとなる。
## 3. メインコンテンツ：左カラム（Time line / アイデアフィード）
画面の約70%を占める、情報の中心地です。
 * **ヘッダー要素:**
   * Time line というタイトルの横に、ソート/フィルタ用のドロップダウンアイコン（v）を配置。
   * **フィルタ機能:** 「Need数（デフォルト：多い順）」「投稿順」など、アイデアの並び順をユーザーが切り替えられるようにする。
 * **スクロール挙動:**
   * **独立スクロール:** サイドバーは固定したまま、このアイデアカードのエリアのみが下にスクロール（または無限スクロール）する。
### アイデアカード（個別コンポーネント）
1行2列のグリッド形式で並ぶカードの詳細です。
 * **基本構成:**
   * **タイトル:** アイデア名 を最上部に太字で配置。
   * **統計:** タイトルのすぐ下に ~ Needs（現在の共感数）を表示。
   * **Goodボタン (:+1:):** カード右上に配置。
     * **機能:** 押すとNeed数がリアルタイムでカウントアップされる。
   * **詳細情報:** 左下に「募集者名」、その下に「#ハッシュタグ的な」タグを配置。
   * **ステータス表示:** 募集者名の横に、進捗状況（未着手・開発中など）を表示する枠を確保。
 * **インタラクション:**
   * **Hover効果:** カードにマウスを乗せると、カード全体が少し浮き上がり（Scale-up）、境界線が強調されるアニメーションを実装。
## 4. メインコンテンツ：右カラム（Community Sidebar）
画面の約30%を占める、コミュニティの熱量を可視化する固定エリアです。
 * **テキストボックス:** 最上部に「サービス概要」や「今週のトピック」などの説明文を配置。
 * **Top Builders アイコン:**
   * **配置:** 3列のグリッドで、最大10人までの円形アイコンを表示。
   * **内容:** 貢献度の高いエンジニアのアバター。
 * **ランキングセクション:**
   * **更新頻度:** 「毎月更新」のラベルを明記。
   * **ランキング1（Top 3）:** タイトルの下に、上位3名の円形アバターアイコンを横並びで配置。
   * **ランキング2 / ランキング3:** 同様に各項目の順位を表示（例：アイデア投稿数ランキング、解決スピードランキングなど）。
## 5. 開発者向けテクニカル・ノート
 1. **State Management:** ナビゲーションの「アクティブ状態」はグローバルなステート（Next.jsならパス名）で管理し、CSSの border-bottom 等で強調する。
 2. **Optimistic UI:** Goodボタンを押した際、DBの更新を待たずに即座にクライアント側の数字を増やし、ユーザーにストレスのない体験（サクサク感）を与える。
 3. **Responsive Design:** モバイル閲覧時は、サイドバー（右カラム）をタイムラインの下に配置するか、あるいはドロワーメニューに格納する設計とする。

# 「NEEDEA」アイデア投稿画面：実装・機能詳細仕様書
## 1. 画面概要
ユーザーの日常にある「不平不満」を、エンジニアが解決可能な「仕様案」へと昇華させるためのインテークフォームです。
 * **画面タイトル:** 日常の『あったらいいな』を世界にポストする
 * **ナビゲーション:** Idea メニューがアクティブ（太字・下線）な状態。
## 2. 入力フォーム構成と詳細仕様
### ① アイデア名 (Title)
 * **UI:** 1行のテキスト入力フィールド。
 * **プレースホルダ:** 例：Excelの集計を自動化したい
 * **機能:** 必須入力。タイムラインで最も目立つ部分になるため、短く簡潔な入力を促す（バリデーション：最大50文字）。
### ② 困り度 (Urgency Level / 1-10)
 * **UI:** ドロップダウンメニュー（v）またはセレクトボックス。
 * **ロジック (注釈の再現):**
   * 選択した数値に応じてUIの色が変化する**カラーフィードバック機能**を実装。
   * 1〜3：青（穏やか）、4〜7：黄（要注意）、8〜10：赤（緊急・切実）。
 * **PM視点:** この数値はタイムラインの「:fire:（Wants）」の初期スコアや表示順位に影響させるアルゴリズムを想定します。
### ③ 日常の不満 (Problem Statement)
 * **UI:** 複数行のテキストエリア。
 * **コンセプト:** 「こんなのがあったらなあ」という理想と、現状の「不満」を繋げる場所。
 * **機能:** 必須入力。ユーザーの「原体験」を記述してもらうことで、解決者の共感を生むエリア。
### ④ 具体的な内容 (Specific Details / Optional)
 * **UI:** 箇条書き形式（項目1、項目2...）の入力フィールド。
 * **機能:** 任意入力だが、注釈通り「書けば書くほど解決される確率が上がる」というマイクロコピーを添える。
 * **実装案:** 最初は2項目分表示し、「＋項目を追加」ボタンで最大5〜10個まで増やせる動的フォーム（Dynamic Form）にするとスマートです。
### ⑤ ハッシュタグ機能 (Tagging)
 * **UI:** 楕円形のタグチップ。
 * **機能:** テキスト入力後に Enter または Space でタグ化。
 * **目的:** カテゴリ分類（#家事 #育児 #仕事）や、想定技術（#Python #React）の紐付け。
## 3. アクションエリア
### ポストするボタン (Submit Button)
 * **UI:** 画面下部に大きく配置されたプライマリボタン。
 * **機能:** * 必須項目（タイトル・不満）が未入力の場合は disabled（無効化）またはエラー表示。
   * 押下後、紙飛行機が飛ぶようなアニメーションと共に「世界へポストされました！」というトースト通知を表示。
   * 投稿完了後、自動的にそのアイデアの「詳細画面（Blueprint）」へリダイレクト。
